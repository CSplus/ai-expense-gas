# 開発ルール

## 1. この文書について

本書は、AI経費処理システムの開発時に守る最低限のルールを定義する。

詳細なコーディング規約やリリース手順は今後整備する。

## 2. 基本方針

* 設計書を優先し、仕様変更時は必要に応じて docs を更新する。
* 現行GWS版の既存動作を壊さないことを優先する。
* Google Apps Script 固有の実装は、可能な範囲で責務ごとにファイルを分ける。
* 共通データモデルと会計ソフト固有仕様を混在させない。
* 会社固有ルールは、コードやAIプロンプトへ固定的に埋め込まない。

## 3. JavaScript / Apps Script の注意点

* グローバル関数名の重複を避ける。
* 経費台帳への書き込みは、可能な範囲で `Sheet.js` のヘルパーに集約する。
* 税率・消費税額の補完処理は `Tax.js` を正とする。
* `COL` の列定義は既存GWS版の互換性維持のため慎重に扱う。
