# Geminiプロンプト

## 1. この文書について

本書は、AI経費処理システムのGWS版で利用する Gemini プロンプトの設計方針を整理する。

現行GWS版では、実際のプロンプト本文は `Gemini.js` の `getReceiptAnalysisPrompt()` に実装している。

## 2. 現行GWS版の対象

現行GWS版では、領収書・レシート画像から以下の情報を抽出する。

* 取引日
* 店舗名または取引先名
* 税込金額
* 経費区分候補
* 支払方法
* インボイス登録番号
* インボイス判定
* 税率
* 消費税額
* 税額判断理由

## 3. 基本方針

* Gemini は経費処理の入力支援を行う。
* Gemini の判断結果は確定情報ではなく、利用者確認を前提とする。
* 会社固有の会計処理ルールは、プロンプトへ固定的に埋め込まない。
* 会計ソフト固有の出力形式やコード変換は、Gemini ではなく連携処理側で扱う。

## 4. 未実装・今後対応

以下は未実装または今後対応とする。

* プロンプト本文のドキュメント正本化
* プロンプトのバージョン管理
* プロンプト変更時のテスト手順
* AI出力の妥当性評価基準
